Notes from Sam regarding the implementation up to Milestone 1 progress.

**Open questions**
- How can we handle pre-existing evidence? For example, some users may have already calculated DMEs, GO enrichment, etc.
- Should we require the user to provide a dataset description? For example, what species, what tissue, treatment, disease, etc, anything relevant? 
- We should update the EnrichR part to match the Serpentine DEG snakemake so it avoids the network dependency issue.
- Can we use module preservation testing (NetRep) as another level of evidence?
- For the non-hdWGCNA case, can we provide different data structures as input? SingleCellExperiment, Seurat Object, or simply a .mtx + matching cell-level meta-data file?
	- We should also require clusters
	- I imagine a non-hdWGCNA case also where the user provides their single-cell data, a set of gene lists (modules) that they wish to score, but they have not yet calculated scores for their list of gene modules. We should allow the user to calculate scores with UCell or with decoupleR, and ideally we should expose as much of the options for those programs to the user as possible.
- How should we handle Pseudo-bulking? Using some of the same scripts as the SERPENTINE to make a SummarizedExperiment?
	- Which pseudobulk DE approach, DEseq2? Just wilcoxon on pseudobulks?
- All tabular outputs (DMEs, gene set overlaps, etc), should be saved for the user to check later!
- Also, we need to make this whole thing compatible with pkgdown!
